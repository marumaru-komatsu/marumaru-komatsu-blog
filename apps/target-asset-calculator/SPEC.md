# 目標資産計算アプリ 仕様書（v0.1）

FIRE / Side-FIRE の「目標資産額」と「資産構成」を、月の生活費・事業所得と
現在の金融資産（NISA・特定・現金）から計算するアプリの仕様。

この文書は、Claude Code / Codex など複数のAIコーディングエージェントに
**同じ正典（source of truth）**として渡すために書く。実装者（AI）は本仕様と
付属の `tests/test_golden.py`（受け入れ基準＝ゴールデン値）を満たすこと。

参照実装（数値エンジンの正典）：同梱の `swr_simulator.py`（モンテカルロSWR計算、検証済み）。
アプリはこのエンジンを再利用または再実装してよいが、**テストを緑に保つこと**。

---

## 1. 目的・スコープ

- **目的**：ユーザーの「月の生活費・事業所得」と「NISA・特定・現金の残高」から、
  (1) いま必要な年間取崩額、(2) 現在資産での達成確度、(3) 目標資産額（見立て別のレンジ）、
  (4) 推奨する資産構成（現金→債券など）を提示する。
- **スコープ内**：上記の計算、感度（株リターン・成功率）、結果の表・グラフ表示。
- **スコープ外**：個別銘柄推奨、税の厳密申告計算、為替の動的モデル、自動売買。
- **非推奨明示**：投資助言ではない。結果は確率であって保証ではない旨を必ず表示する。

---

## 2. モデル前提（既定値・固定ロジック）

すべて**実質（インフレ調整後・今日の円）ベース**で扱う。

| 前提 | 既定値 | 備考・固定ルール |
| --- | --- | --- |
| インフレ率 | 2% | 取崩額・生活費は実質固定として扱う |
| 株式リターン | 幾何実質 5%（中心）/ 感度 5・6・7% | **幾何平均**（複利で実際に増える率）として入力 |
| 株式ボラティリティ | 18%（対数リターンの年率SD） | |
| 債券リターン | 幾何実質 +2.5%、ボラ 6% | 「防御資産＝債券」を選んだ場合 |
| 現金リターン | 幾何実質 −2%、ボラ 0% | 「防御資産＝現金」を選んだ場合（名目0%−インフレ2%） |
| 株と防御資産の相関 | 0（独立） | |
| リバランス | 年次（年初に目標比率へ戻す） | 1年グロス＝各資産グロスの**加重算術平均** |
| 取崩タイミング | 期初（毎年の頭で取り崩す） | 保守側 |
| 取崩額 | 実質固定（初年度額を毎年維持） | 4%ルールと同じ「実質一定取崩」 |
| 取崩期間 | 30年（既定） | 以降は公的年金等でブリッジする想定 |
| 判定 | 30年の途中で資産が尽きなければ「成功」 | |
| 乱数 | seed・nsims を固定して再現可能に | 既定 seed=2026, nsims=60000（テストは40000） |

> **重要な不変条件**：
> 1. 「幾何平均アンカー」を崩さない（算術平均で入れると volatility drag を二重計上し過小評価になる）。
> 2. 年次リバランスは**加重算術平均** `w_stock*exp(stock_logr)+w_def*exp(def_logr)`。加重幾何平均にしない。
> 3. 安全取崩率の二分探索は、**乱数パスを探索前に一度だけ生成**して全判定で共通化する（成功率関数を rate に単調にするため）。

## 3. 入力

| 入力 | 型 | 例 | 必須 | 備考 |
| --- | --- | --- | --- | --- |
| 月の生活費 | 万円/月 | 18 | ✓ | 税・社保込みか別かを選べると尚可（既定：込み） |
| 月の事業所得 | 万円/月 | 5 | ✓ | Side-FIREの下限想定。0ならフルFIRE |
| NISA残高 | 万円 | 1,800 | ✓ | 株式（非課税） |
| 特定口座残高 | 万円 | 1,500 | ✓ | 株式（課税：譲渡益20.315%） |
| 現金残高 | 万円 | 825 | ✓ | 防御資産 |
| 防御資産の種類 | enum | cash / bond | | 既定 cash。bond で実質+2.5% |
| 株式の期待リターン（実質・幾何） | % | 5 | | 既定5、感度で5/6/7 |
| 目標成功率 | % | 90 | | 80/90/95 から選択 |
| 取崩期間 | 年 | 30 | | 既定30 |
| 含み益割合（特定口座） | % | 50 | | 税グロスアップ用の仮定 |

## 4. 出力

1. **年間取崩額（税抜き）**：`(月生活費 − 月事業所得) × 12`。例：(18−5)×12 = **156万円/年**。
2. **税グロスアップ後の実効取崩額**：特定口座の譲渡益課税を織り込んだ年額。例：**約162万円/年**。
3. **現在資産での達成確度**：いまの総資産で実効取崩を賄える成功率（%）。
4. **目標資産額レンジ**：成功率別（80/90/95%）×株リターン別（5/6/7%）の必要額テーブル。
5. **推奨アクション**：現金→債券の置換効果、確度を上げるのに必要な追加額など。
6. **感度**：株リターン・成功率・防御資産の種類/比率を変えたときの必要額変化。

各出力に**前提（株リターン・成功率・防御資産）**を必ず併記する。

## 5. 計算ロジックの定義（実装者向け）

### 5.1 取崩額
```
annual_withdrawal(net)  = (monthly_living - monthly_side_income) * 12        # 万円/年
```

### 5.2 税グロスアップ（特定口座の譲渡益課税）
本アプリでは「特定口座を年取崩で割って課税フェーズ比率を出し、ブレンドする」簡易法を採用する
（厳密法ではない旨を表示）。
```
years_taxable      = taxable_balance / annual_withdrawal_net      # 特定で賄える年数
taxable_phase_frac = min(years_taxable / years, 1)               # 課税フェーズの割合
per_year_uplift    = 1/(1 - gain_fraction * 0.20315) - 1         # 特定のみ取崩時の上乗せ率
blended_uplift     = taxable_phase_frac * per_year_uplift
annual_withdrawal_gross = annual_withdrawal_net * (1 + blended_uplift)
```
既定（taxable=1500, net=156, years=30, gain=0.5）で `blended_uplift ≈ 3.62%`、
`gross ≈ 161.7万円`。

### 5.3 資産構成（残高→比率）
```
stock_balance     = nisa + taxable          # NISA・特定は株式
defensive_balance = cash                    # 現金（または債券に置換）
total             = stock_balance + defensive_balance
stock_weight      = stock_balance / total
defensive_weight  = defensive_balance / total
```
例：(1800+1500)/4125 = **0.8**、825/4125 = **0.2**。

### 5.4 安全取崩率・必要額・成功率
参照エンジン `swr_simulator.py` の以下を用いる（同義の再実装可）：
- `safe_withdrawal_rate(stock_geo, stock_vol, def_geo, def_vol, def_weight, target, years, nsims, seed)`
- `required_asset(...) -> (swr, withdrawal/swr)`
- `_success_given_gross(rate, gross)` 相当（現在資産での成功率＝`gross`生成後に rate=gross_withdrawal/current_total で評価）

恒等式：`required_asset = annual_withdrawal_gross / safe_withdrawal_rate`。

## 6. 受け入れ基準（Acceptance Criteria）

`tests/test_golden.py` を**すべて緑**にすること。基準は2層。

### 6.1 性質テスト（実装非依存・許容幅つき）
どんな正しい実装でも満たすべき不変条件。
- **校正**：古典PF（株75/債25・実質株7%/債2.5%）で、90%成功のSWRが **4.0〜4.6%** に入る（＝「4%ルール」を再現）。
- **単調性**：取崩率を上げると成功率は下がる（`success(3%) > success(5%)`）。
- **方向性**：同じ防御比率なら、債券の必要額 ≤ 現金の必要額（90%成功）。
- **恒等式**：`required_asset == withdrawal / swr`（相対1e-6）。
- **決定性**：防御0%（株100%）は cash/bond どちらでも同一。同一 seed/nsims で再現一致。
- **エッジ**：現金0%・株100%、事業所得≥生活費（取崩0以下→例外 or 0扱い）、極端な取崩率で破綻しない。

## 7. 非機能要件

- **再現性**：同一入力＋seed＋nsimsで同一出力。乱数は明示的に seed 管理。
- **性能**：1ケース（nsims=40000）を数百ms〜1s程度。スイープ表は並列化可。
- **数値の隔離**：数値ロジックは純粋関数モジュール（`fire_app` / `engine`）に置き、UIと分離。
- **免責表示**：全画面に「投資助言ではない／自己責任」を明示。
- **エラー処理**：取崩額≤0（事業所得≥生活費）、残高0、比率範囲外を明確に扱う。

## 9. 用語

- **SWR（安全取崩率）**：指定成功率を満たす最大の初年度取崩率。
- **成功率**：30年の途中で資産が尽きなかったシナリオの割合。
- **幾何平均**：複利で実際に増える率（＝中央値パスの成長率）。
- **ボラティリティ・ドラッグ**：値動きにより幾何平均が算術平均より `≈ボラ²/2` 下がる現象。
- **防御資産**：現金または債券（株式の値動きを抑える枠）。
